Building aic — open-source dev tools · Rust
2h ·
I've shipped a lot of commits. And for years, the most tedious part of every single one was the same: writing the message. Not because it's hard — because it breaks flow. You finish a clean change, switch to your terminal, and now you're summarizing 400 lines of diff into a one-liner. So I built aic — an open-source CLI that reads your staged diff and drafts a clean, conventional commit message for you. One command. How it works: → Stage your work (or don't — see below) → Run aic → It reads the diff, drafts the message, commits Three commands, total: • aic — commit staged work, or auto-batch unstaged work into logical commits • aic setup — one-time wizard: provider · API key · model • aic list — inspect resolved config A few things I cared about: ✦ No lock-in — works with OpenAI, Anthropic, Gemini, DeepSeek, Groq, and Ollama ✦ Your key, your model — no per-commit markup, no middleman; it runs locally and calls your provider directly ✦ Smart batching — if you forgot to stage, aic groups unstaged changes into multiple logical commits instead of one messy dump ✦ Built in Rust — a single, fast binary Install: brew install aic (or: cargo install aic / one-line installer in the README) MIT-licensed, and v0.1.6 is the first signed release. If you ship a lot of commits, I'd love your feedback: github.com/CaicoLeung/aic #rustlang #developerexperience #devtools #opensource #git
